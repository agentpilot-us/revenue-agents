# Content Library outline — StradexAI / AgentPilot

Use this to plan **Content Library** items that power **AI-generated outreach** and **play steps**. The app’s `ContentType` enum (`prisma/schema.prisma`) includes more types than the primary “+ Add Content” wizard; this outline covers **all** types and marks **UI-create** paths where applicable.

**Positioning anchor:** StradexAI sells **managed AI-powered growth marketing and account execution** (intelligence, signals, plays, governed content). **AgentPilot** is the **delivery engine** you operate—clients often receive **outputs** through existing workflows, not a mandatory new rep tool. Library copy should reinforce **service outcomes**, **governance**, **speed**, and **portfolio coverage**, not only “log in to our platform.”

## 2. Suggested inventory by type (outline)

### 2.1 Framework (add 3–6 items)

Focus on **how you work**, not only software features.

1. **Engagement model: managed execution vs. software rollout** — Default delivery through email/CRM/Slack; optional deployment when approved.  
2. **The 70/30 split** — You handle research, signal triage, mapping, drafting; AE owns judgment, relationships, deal strategy.  
3. **Signal-to-play operating model** — Triage SLA, Draft + Review, autonomy levels, when human approves send.  
4. **Weekly account briefing cadence** — What “Monday AM briefing” contains; how priorities are ranked.  
5. **Governance layer** — Brand voice, competitive claims, discount/cooldown rules, cadence limits.  
6. **90-day pilot structure** — Phase 1 setup vs. Phase 2 retainer; success metrics (signal response, play completion, content acceptance, time reclaimed).

### 2.2 Use Case (keep 6–10; refresh titles/descriptions)

Reframe from “AE uses the app” to “**StradexAI + AgentPilot delivers X to the account team**.”

| Theme | Example title angle |
|-------|---------------------|
| Portfolio coverage | “Full named-account coverage without adding headcount” |
| Signal speed | “High-value signals triaged and drafted within 48 hours” |
| Buying committee | “Buying-group mapping and multi-threading as a managed deliverable” |
| Governance at scale | “On-brand, compliant outreach at volume (Draft + Review)” |
| RevOps / enablement | “Single partner for intelligence + plays + content (not six tools)” |
| Security / procurement | “Low footprint delivery; optional approved technical deployment” |
| SKO / pipeline moments | “Post-SKO execution: plays and content ready for strategic accounts” |

Retire or rewrite anything that implies **every AE must adopt AgentPilot as a daily UI** unless that’s true for a specific segment.

### 2.3 Success Story (3–8; anonymize unless cleared)

Structure: **Situation → what we operated → metric → quote (optional).**

Suggested arcs:

- **Signal → meeting** in days (exec hire, partnership news, earnings).  
- **Portfolio activation** — % of named accounts with an active play / briefing in a quarter.  
- **Time reclaimed** — AE self-report or activity proxy (hours/week back to selling).  
- **Content acceptance** — % of drafts sent with minor edits.  
- **Multi-thread** — net-new contacts engaged in a division in N weeks.  

Avoid implying **NVIDIA-specific** details in titles unless public and approved.

### 2.4 CompanyEvent (“Event” in UI) (2–6)

- **Industry / ecosystem** events (SaaS, data, manufacturing tech) where you offer **briefings or working sessions**.  
- **Executive roundtables** or **private dinners** (if applicable).  
- **Webinar**: “Managed ABX execution in 90 days” (methodology, not feature dump).  
- Update **Design Partner** copy if you still use it: clarify **partnership on delivery + roadmap** vs. “free product trial” if the economics changed.

### 2.5 FeatureRelease (3–8)

Reframe releases as **client-visible capabilities** of the **service**:

- **Signal-to-play mapping** / priority rules.  
- **Buying group intelligence** outputs in briefings.  
- **Local / private deployment option** (if true).  
- **Governance**: new controls (cooldowns, claim rules, approval paths).  
- **New channel outputs** (e.g. LinkedIn post drafts, exec briefing format).

Each item: **one sentence outcome** + **who benefits** (AE, RevOps, leadership).

### 2.6 ResourceLink (4–10)

Short description + URL for:

- **Service overview / one-pager** (PDF).  
- **90-day pilot / SOW sample** (if shareable).  
- **Security / data handling FAQ**.  
- **ROI / comparison** (vs. SDR, vs. agency)—align with SOW table if used.  
- **Calendly / contact** for scoping call.

### 2.7 Battlecard (3–6 competitors / alternatives)

Rows/alternatives to address:

| Alternative | Your wedge |
|-------------|------------|
| Hire SDRs | Ramp time, variable quality, no built-in governance/plays. |
| ABM agency | Campaign bursts vs. **always-on signal + play** capacity. |
| Point intelligence tools | Insights without **execution** and **governed content**. |
| Generic AI / ChatGPT | No account context, no play orchestration, no compliance story. |
| “We’ll build it in-house” | Cost, maintenance, model churn, **time to value**. |

### 2.8 Persona (3–5)

One item per primary buyer/influencer:

- **CRO / VP Sales** — pipeline, cost per opportunity, AE productivity.  
- **Director RevOps / Sales Ops** — workflow, CRM alignment, metrics, tool sprawl.  
- **IT / Security** — data handling, deployment options, audit language.  
- **Enterprise AE** (user-champion) — time savings, quality of drafts, signal speed.

### 2.9 UploadedDocument (1–3)

- Anonymized **SOW** or **statement of work template** (PDF).  
- **Master services description** + **deliverables matrix**.  
- **SOC 2 / security** summary if you have one.

### 2.10 VideoLink (0–2)

- **3-minute**: “What you receive each week as a strategic AE.”  
- **Founder**: why **managed + AI** vs. **tool-only**.

## 3. Coverage vs. active plays (how to think about it)

The health panel suggests **content types** per **play template step**—at **type** level, aim for:

| Play pattern | Especially helpful types |
|--------------|---------------------------|
| **Signal / news response** | UseCase (signal speed), SuccessStory, Framework (governance), FeatureRelease (signal mapping). |
| **Executive / new hire** | SuccessStory, Persona (economic buyer), EmailContent snippets if used. |
| **Expansion / cross-sell** | UseCase (multi-thread), Battlecard, SuccessStory. |
| **Competitive** | Battlecard, Framework (approved claims). |
| **Event / invite** | CompanyEvent, SuccessStory (event-driven pipeline). |
| **Renewal / QBR-style** | Framework (value narrative), SuccessStory, ResourceLink (ROI). |

Keep **at least one strong Framework** and **2+ SuccessStories** so multi-step plays have proof and methodology to pull from.
